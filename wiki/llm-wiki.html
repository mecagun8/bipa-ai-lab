왜 LLM-Wiki가 필요한가

RAG는 질문할 때마다 원문에서 다시 검색해요. 그러다 보니 지식이 쌓이지 않고, 비용도 매번 발생해요. 반면 LLM-Wiki는 LLM이 직접 마크다운 위키를 작성하고 관리해요. 새 문서가 들어오면 관련 페이지를 업데이트하고, 시간이 갈수록 지식이 복리로 쌓여요.

💬
Andrej Karpathy (전 Tesla AI 디렉터)
"나는 요즘 코드보다 개인 지식 저장소 구축에 토큰을 더 많이 쓰고 있다. RAG가 매번 원문에서 재추출하는 동안, LLM-Wiki는 지식이 점진적으로 축적되는 영속적 위키를 구축한다. 위키 유지 관리 비용이 거의 0에 수렴하는 게 핵심이다."

이런 상황에 쓰세요

상황LLM-Wiki가 맞는 이유
내부 용어·약어가 많아서 AI가 자꾸 틀림용어집 페이지를 만들어두면 모든 대화에 일관 적용
같은 규정을 반복해서 물어봄한 번 위키에 정리해두면 이후 토큰 비용 없음
여러 문서를 종합해서 판단해야 함LLM이 교차 참조 페이지를 자동 생성
지식을 팀원과 공유하고 싶음마크다운 파일 = git으로 버전 관리·공유 가능
RAG 검색이 자꾸 엉뚱한 청크를 가져옴LLM이 이미 합성한 위키 페이지를 직접 읽음

폴더 구조

마크다운 파일 디렉터리 하나면 충분해요. Claude Code가 이 폴더를 읽고 관리해요.

bipa-wiki/
  sources/        # 원문 소스 (변경 불가, 읽기 전용)
    복무규정_2026.txt
    사업안내_일자리.txt
  wiki/           # LLM이 작성·관리하는 위키
    index.md      # 전체 페이지 목록 + 한줄 요약
    log.md        # 인제스트·쿼리 이력
    용어집_BIPA.md
    복무규정_요약.md
    팀별역할.md
  CLAUDE.md       # Claude Code에게 위키 규칙을 알려주는 파일

레이어 역할 분리
sources/ — 원문. LLM은 읽기만 해요. 진실의 원천이에요.
wiki/ — LLM이 작성·관리. 사람은 읽기만 해도 돼요.
CLAUDE.md — 위키 구조와 규칙을 Claude Code에게 알려주는 설정 파일이에요.

CLAUDE.md 작성

이 파일이 LLM-Wiki의 핵심이에요. Claude Code가 처음 실행될 때 읽고 위키 관리 방식을 파악해요.

# BIPA AI Lab — LLM Wiki 관리 규칙

## 역할
당신은 BIPA 인재양성단의 내부 지식 위키 관리자입니다.
sources/ 폴더의 원문을 읽고 wiki/ 폴더의 마크다운 페이지를 작성·유지합니다.

## 규칙
- sources/ 파일은 절대 수정하지 않습니다 (읽기 전용)
- 새 소스 인제스트 시 관련 wiki 페이지를 모두 업데이트합니다
- 모든 페이지는 wiki/index.md에 등록합니다
- 작업마다 wiki/log.md에 기록합니다

## 내부 용어
- BIPA: 부산정보산업진흥원
- 인재양성단: 교육훈련 담당 부서
- (추가 용어는 wiki/용어집_BIPA.md 참조)

## 명령어
- "인제스트: [파일명]" → 해당 소스를 읽고 위키에 통합
- "쿼리: [질문]" → 위키를 검색해 답변
- "린트" → 위키 상태 점검 (모순, 고아 페이지 등)

실제 운영 방법

1
초기 설정
폴더 구조 생성 후 CLAUDE.md 작성해요.
Claude Code로 해당 폴더를 열고 "위키를 시작합니다. CLAUDE.md를 읽고 초기 구조를 만들어줘"라고 하면 index.md, log.md를 자동 생성해요.
2
문서 인제스트 (새 문서 추가)
원문을 sources/에 넣고 Claude Code에게 지시해요.
"인제스트: 복무규정_2026.txt"
→ LLM이 읽고 → 위키 페이지 작성 → index.md 업데이트 → log.md 기록
단일 소스가 10~15개 페이지에 영향을 줄 수 있어요.
3
질문 (쿼리)
"쿼리: 연차 휴가 신청 절차가 어떻게 돼?"
→ Claude Code가 index.md 먼저 확인 → 관련 페이지 탐색 → 답변 합성
좋은 답변은 위키에 새 페이지로 저장할 수도 있어요.
4
린트 (주기적 정리)
"린트" 명령으로 위키 상태를 점검해요.
모순 페이지, 오래된 내용, 연결이 없는 고아 페이지를 찾아 정리해줘요.
