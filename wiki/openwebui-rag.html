AI에 문서를 연결하는 두 가지 방법

업무 문서를 AI가 참조하게 만드는 방법은 두 가지예요. 이 가이드는 RAG(Open WebUI Knowledge) 방법을 다뤄요.

이 가이드
📂 RAG — Open WebUI Knowledge
파일을 올리면 AI가 질문마다 원문 텍스트 조각을 검색해서 가져와요. 검색 엔진에 가까워요.

이럴 때 쓰세요
· "몇 조 몇 항" 정확한 출처·원문이 필요할 때
· 문서가 자주 바뀌어서 항상 최신 원문이어야 할 때
별도 가이드
📖 LLM-Wiki — Claude Code
AI가 원문을 읽고 직접 이해해서 위키 페이지를 써요. 이후엔 그 위키를 참조해요.

이럴 때 쓰세요
· 용어·개념·절차 설명이 필요할 때
· 여러 문서를 종합해서 답해야 할 때
→ LLM-Wiki 가이드 보기
📌
같은 질문, 다른 방식
"연차 신청 어떻게 해?"
· RAG → 복무규정.txt에서 관련 문장 조각을 뽑아옴
· LLM-Wiki → 위키에 이미 정리된 연차 절차 페이지를 읽고 답변

"BIPA가 뭐야?"
· RAG → 문서에서 "BIPA"가 언급된 문장을 찾아옴
· LLM-Wiki → 용어집 페이지에서 정의·맥락을 통째로 답변

RAG를 써야 하는 상황

상황RAG가 맞는 이유
규정집·지침서 원문 검색"몇 조 몇 항" 같은 정확한 출처가 필요할 때
문서가 자주 업데이트됨파일만 교체하면 바로 반영됨
대용량 문서 일회성 분석위키로 만들 필요 없이 업로드 후 바로 질문
팀원이 직접 문서 관리파일 업로드만 알면 됨, 별도 설정 불필요

가장 흔한 실수 — 통파일 업로드

규정집 300페이지 PDF를 그대로 올리면 검색 정확도가 크게 떨어져요.

❌ 잘못된 방법 — 통파일
📄 규정집_전체.pdf (300p)
→ 관련 없는 내용이 섞여 검색 정확도 낮음
✅ 올바른 방법 — 주제별 분리
📝 복무규정.txt
📝 출장규정.txt
📝 교육비지원.txt
📝 인사규정.txt
⚠️
한국어 청크 기준: 1,500~2,000자
RAG는 문서를 청크(조각)로 잘라 검색해요. 한 파일이 너무 크면 엉뚱한 청크가 검색돼요. A4 기준 약 1~3페이지 분량이 최적이에요.

문서 준비

1
TXT로 변환
PDF·HWP 내용을 메모장에 복사·붙여넣기해요. 표·그림은 제거하고 텍스트만 남겨요.
2
주제별로 파일 분리
하나의 파일 = 하나의 주제. 규정.txt ❌ → 2026_복무규정_연차휴가.txt ✅
3
용어집 별도 작성
"BIPA", "인재양성단" 같은 내부 용어는 AI가 모를 수 있어요.
용어집_BIPA.txt에 "인재양성단: BIPA 내 교육훈련 담당 부서" 형식으로 정리해요.

Open WebUI에 등록

1
Knowledge 메뉴 접속
좌측 사이드바 → Workspace → Knowledge → + New Knowledge
2
TXT 파일 업로드
준비한 파일들을 드래그 앤 드롭. 자동으로 청크 분할돼요.
3
모델에 연결
Workspace → Models → 모델 편집 → Knowledge 섹션에서 연결 → Save
4
채팅에서 활성화
채팅창에서 # 입력 → Knowledge 선택 → RAG 활성화됨

확인

테스트기대 결과
등록 문서에만 있는 내용 질문문서 기반으로 정확한 답변
"어디서 찾았어?" 질문파일명과 출처 표시
검색이 잘 안 된다면파일을 더 작게 쪼개거나 Settings → RAG → Chunk Size를 1500으로 낮추기
💡
내부 용어가 계속 틀리게 해석된다면?
RAG 한계예요. 이 경우 LLM-Wiki 방식으로 용어집을 구조화하는 게 더 효과적이에요.
