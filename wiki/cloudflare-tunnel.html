Cloudflare Quick Tunnel로
Ollama 팀원에게 공유하기

⚙️ 인프라 · 네트워크 ⏱ 소요시간 5분 📅 2026.05

개요

Cloudflare Quick Tunnel은 계정 없이 명령어 하나로 로컬에서 실행 중인 서비스를 외부에서 접속 가능한 임시 URL로 노출하는 방법이에요. PowerShell 창이 열려 있는 동안만 유효한 임시 방식이라 팀원에게 잠깐 공유할 때 적합해요.

⚠️
임시 URL 주의
실행할 때마다 URL이 달라지고, PowerShell 창을 닫으면 즉시 무효화돼요. 고정 URL이 필요하면 Cloudflare 계정 기반 Named Tunnel을 사용해야 해요.

사전 준비

항목상태확인 방법
Ollama 실행 중필수트레이 아이콘 확인
Open WebUI 실행 중필수localhost:3000 접속
cloudflared 설치필수아래 설치 단계 참고
Cloudflare 계정불필요—

1단계 · cloudflared 설치

아래 두 가지 방법 중 하나로 설치해요.

A
winget으로 설치 (권장)
PowerShell 관리자 모드에서 실행
PowerShell
winget install Cloudflare.cloudflared
B
직접 다운로드
github.com/cloudflare/cloudflared/releases/latest → cloudflared-windows-amd64.msi 다운로드 후 설치

2단계 · 서비스 실행 확인

터널을 열기 전에 Open WebUI가 정상 작동 중인지 확인해요.

PowerShell
# Ollama 모델 목록 확인
ollama list

# 브라우저에서 Open WebUI 확인
→ http://localhost:3000 접속되면 OK

3단계 · Quick Tunnel 실행

PowerShell에 아래 명령어를 입력해요. 10~30초 후 URL이 생성돼요.

PowerShell
cloudflared tunnel --url http://localhost:3000

성공하면 이런 로그가 나타나요:

출력 예시
INF Thank you for trying Cloudflare Tunnel...
INF Requesting new quick Tunnel on trycloudflare.com...
INF Your quick Tunnel has been created!
INF Visit it at: https://registrar-greene-hugo-tigers.trycloudflare.com

4단계 · URL 공유

생성된 .trycloudflare.com URL을 카카오톡, 슬랙 등으로 팀원에게 공유하면 바로 접속 가능해요.

💡
동시 접속 안내
RTX 2070 Super 기준 동시에 여러 명이 사용하면 응답이 느려져요. 시연 목적이면 1~2명씩 순차 접속을 권장해요.

자주 하는 실수

✗
Ollama / Open WebUI 미실행 상태에서 터널 실행
접속해도 연결 오류 화면만 나와요. 반드시 서비스 먼저 확인하세요.
✗
PowerShell 창을 닫음
창을 닫는 순간 URL이 즉시 무효화돼요.
✗
포트 번호 오류
Open WebUI 포트가 3000이 아니면 --url http://localhost:포트번호로 수정해야 해요.
